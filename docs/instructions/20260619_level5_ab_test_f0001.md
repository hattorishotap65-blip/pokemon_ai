# Level 5 Instruction: A/B Test for F0001

## Purpose

Level 5 では、Level 4 で生成した修正候補を baseline / candidate で比較し、改善したかを判定できる状態にする。

今回は **F0001: voltorb_over_wattrel_missed** のみを対象にする。

## Scope

今回やること:

- F0001 の baseline を保存する
- F0001 だけを改善する最小修正を行う
- candidate のシミュレーションを実行する
- anomaly 検知を再実行する
- baseline と candidate を比較する
- 改善 / 保留 / 悪化 を判定する
- 結果を `reports/level5_ab_test_f0001.md` と `reports/level5_ab_test_f0001.json` に保存する

今回やらないこと:

- F0002 / F0003 / F0004 の修正
- deck.csv の変更
- 重み自動探索
- 自動PR作成
- 大規模リファクタリング
- 複数候補の同時修正

## Target Candidate

```text
F0001: voltorb_over_wattrel_missed
```

問題:

```text
Wattrel で攻撃しているが、Voltorb に交代して攻撃した方が打点が高い可能性がある。
```

期待する修正方針:

```text
Wattrel 攻撃よりも、Voltorb へ交代して攻撃する方が明確に高打点の場合だけ、Voltorb への retreat / switch を高く評価する。
```

## A/B Metrics

必ず比較する指標:

- `error_rate`
- `fallback_rate`
- `avg_decision_time_ms`
- `p95_decision_time_ms`
- `attack_available_but_no_attack`
- `end_when_attack_available`
- `retreat_when_attack_available`

- `best_damage_attacker_not_selected`
- `voltorb_attack_count`
- `wattrel_attack_count`
- `retreat_to_voltorb_count`
- `win_rate`

## Acceptance Criteria

合格条件:

- `error_rate` が悪化しない
- `fallback_rate` が悪化しない
- `p95_decision_time_ms` が大きく悪化しない
- `attack_available_but_no_attack` が 0 のまま
- `end_when_attack_available` が 0 のまま
- `retreat_when_attack_available` が 0 のまま
- `voltorb_over_wattrel_missed` が減る
- `best_damage_attacker_not_selected` が増えない
- `win_rate` が悪化しない

保留条件:

- F0001 は減ったが F0002 / F0003 が増える
- Voltorb 優先により Bellibolt ex の妥当攻撃まで減る
- retreat が増えすぎる
- win_rate が悪化するが異常行動だけは減る

不合格条件:

- critical / high anomaly が増える
- error_rate / fallback_rate が悪化する
- 攻撃可能なのに End / Retreat するケースが発生する
- F0001 は減らず、他の異常だけ増える

## Verification Outline

1. `reports/baseline/` と `reports/candidate/` を作成する
2. baseline の anomaly report を保存する
3. F0001 だけの修正を行う
4. candidate simulation を実行する

7. 結果を `reports/level5_ab_test_f0001.md` / `.json` に保存する
8. agent behavior files を変更した場合は `python build_submission.py` を実行する
